**SEO técnico para Next.js** é o conjunto de implementações no código — metadados,
dados estruturados, sitemaps, renderização e performance — que permite ao Google
rastrear, indexar e rankear um site feito com App Router. Diferente do SEO de
conteúdo, ele é resolvido no repositório: `Metadata API`, `sitemap.ts`,
`robots.ts`, JSON-LD e Core Web Vitals. Este guia mostra como implementar cada
peça, com código real e resultados medidos neste próprio site.

## O que este guia vai cobrir

> **Guia em expansão.** Este é o mapa do conteúdo em produção. Cada tópico
> abaixo vira um capítulo com artigo aprofundado, código e medições reais —
> publicados nas próximas semanas.

1. **Metadata API do Next.js** — title, canonical e Open Graph na prática
2. **JSON-LD no App Router** — [como implementar JSON-LD no Next.js derivando o schema do frontmatter](/blog/json-ld-nextjs)
3. **sitemap.ts e robots.ts** — sitemaps dinâmicos nativos
4. **SSR vs SSG vs ISR** — o que o Googlebot realmente enxerga
5. **Core Web Vitals: LCP** — [como melhorar o LCP no Next.js pelas 4 subpartes](/blog/melhorar-lcp-nextjs) e [os erros que impedem o LCP de melhorar](/blog/lcp-alto-next-js)

7. **next/image e SEO** — boas práticas de imagem
8. **Hreflang no Next.js** — site bilíngue passo a passo
9. **Redirects e canonicals** — next.config vs middleware
10. **GTM em SPA/Next.js** — history change e virtual pageviews

## Por que confiar neste guia

Este site é um laboratório vivo: cada técnica documentada aqui está
implementada neste próprio domínio e medida com Google Search Console, CrUX e
experimentos antes/depois. Nenhuma recomendação entra no guia sem ter sido
testada de verdade.
